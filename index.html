Chapter 1: CSS Introduction

CSS ka full form hai Cascading Style Sheets. CSS ka kaam HTML elements ko style karna hota hai. HTML batata hai page par kya content hai, aur CSS batata hai woh content kaisa dikhega. Example: heading ka color kya hoga, paragraph ka font size kya hoga, button ka background kya hoga, cards side-by-side honge ya one-by-one, mobile par layout kaise adjust hoga, ye sab CSS se hota hai.

Real-world example: HTML ek ghar ka structure hai, aur CSS us ghar ka paint, decoration, furniture arrangement, and lighting hai. Sirf structure se ghar complete nahi lagta. Design add karne ke baad woh usable and attractive lagta hai.

Types of CSS

Type Example Explanation
Inline CSS <h1 style="color: blue;"> CSS direct HTML tag ke andar likhte hain. Small testing ke liye okay, but real projects mein avoid karna better hai.
Internal CSS <style> h1 { color: blue; } </style> CSS same HTML file ke head section mein likhte hain. Single page demo ke liye useful hai.
External CSS <link rel="stylesheet" href="styles.css"> CSS separate file mein likhte hain. Professional projects ke liye best approach hai.

External CSS Practical

<link rel="stylesheet" href="styles.css">
Code Detailed Explanation
<link> External resource connect karne ka HTML tag. CSS file ko HTML page se connect karne ke liye use hota hai.
rel="stylesheet" Browser ko batata hai ki linked file stylesheet hai. Matlab browser is file ko design rules ke form mein read karega.
href="styles.css" CSS file ka path. Agar CSS file same folder mein hai toh direct styles.css likhenge.
Important: Agar CSS file ka naam galat likh diya, jaise file ka naam style.css hai but HTML mein styles.css likha hai, toh CSS apply nahi hogi.

Chapter 2: CSS Syntax

CSS syntax selector, property, and value se milkar banta hai. Selector batata hai kis HTML element ko style karna hai. Property batati hai kaunsi style change karni hai. Value batati hai us property ki setting kya hogi.

Syntax Structure

selector {
  property: value;
}

Example

h1 {
  color: blue;
  font-size: 36px;
}

Line-by-Line Explanation

Code Explanation
h1 Selector hai. Page ke saare h1 headings ko target karega.
{ } CSS block start and end karta hai. Iske andar CSS declarations likhi jaati hain.
color: blue; Text color blue set karta hai. color property hai, blue value hai.
font-size: 36px; Heading ka font size 36 pixels set karta hai.

Chapter 3: Selectors and Specificity

Selectors CSS ka foundation hain. Selector decide karta hai ki CSS rule kis element par apply hoga. Agar selector wrong hai, CSS likhne ke baad bhi style apply nahi hogi.

Common Selectors

Selector Type Code Meaning
Element Selector p Page ke saare paragraph tags target karta hai.
Class Selector .card Jis element par class="card" hoga usko target karta hai.
ID Selector #hero Jis element par id="hero" hoga usko target karta hai.
Descendant Selector nav a Nav ke andar ke anchor links target karta hai.
Pseudo-class a:hover Mouse hover hone par style apply karta hai.

Specificity Kya Hai?

Specificity CSS priority system hai. Agar same element par multiple CSS rules apply ho rahe hain, toh browser stronger selector choose karta hai. Simple priority: inline style sabse strong, phir ID selector, phir class selector, phir element selector.

p {
  color: black;
}

.text {
  color: blue;
}

#intro {
  color: red;
}

Agar ek paragraph par class="text" and id="intro" dono hain, toh red color apply hoga because ID selector class selector se stronger hota hai.

Chapter 4: Box Model and Layout Concepts

CSS box model ke according har HTML element ek box hota hai. Har box ke four parts hote hain: content, padding, border, and margin. Spacing and layout samajhne ke liye box model very important hai.

Part Meaning
Content Actual text ya image area.
Padding Content and border ke beech inside space.
Border Element ke around line.
Margin Element ke bahar outside space.

Practical Code

.service-card {
  background: white;
  padding: 24px;
  border: 1px solid #e2e8f0;
  margin-bottom: 20px;
  border-radius: 8px;
}
Code Explanation
background: white; Card ka background white banata hai.
padding: 24px; Card ke andar content ke around space create karta hai.
border Card ke around light border create karta hai.
margin-bottom Card ke bahar neeche space add karta hai.
border-radius Card ke corners rounded banata hai.

Chapter 5: Flexbox Layout System

Flexbox one-dimensional layout system hai. Iska use row ya column direction mein items align karne ke liye hota hai. Navbar, buttons group, hero actions, and simple alignment ke liye Flexbox best hai.

.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  gap: 24px;
}
Code Explanation
display: flex; Container ke children ko flex layout mein arrange karta hai.
justify-content: space-between; Items ko horizontal direction mein left and right side spread karta hai.
align-items: center; Items ko vertical center align karta hai.
gap: 24px; Items ke beech fixed spacing add karta hai.

Chapter 6: CSS Grid Layout System

CSS Grid two-dimensional layout system hai. Grid rows and columns dono control kar sakta hai. Cards, galleries, services section, dashboard, and page layout ke liye Grid powerful hai.

.services-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 24px;
}
Code Explanation
display: grid; Container ko grid layout mein convert karta hai.
grid-template-columns: repeat(3, 1fr); 3 equal columns create karta hai. 1fr available space ka equal fraction hai.
gap: 24px; Cards ke beech spacing add karta hai.

Chapter 7: Responsive Design Principles

Responsive design ka matlab website different screen sizes par properly work kare. Desktop, tablet, and mobile par website readable and usable honi chahiye. Responsive design ke liye flexible width, responsive images, readable typography, and media queries use hote hain.

img {
  max-width: 100%;
  height: auto;
}

.container {
  width: min(1120px, 92%);
  margin: 0 auto;
}
Code Explanation
max-width: 100%; Image parent container se bahar overflow nahi karegi.
height: auto; Image ka original ratio maintain karta hai.
width: min(1120px, 92%); Desktop par max 1120px, mobile par 92% width use karta hai.

Chapter 8: Media Queries

Media query CSS ka rule hai jo screen size ke according styles change karta hai. Example: desktop par 3 columns, mobile par 1 column. Responsive design mein media queries very important hain.

@media (max-width: 768px) {
  .navbar {
    flex-direction: column;
    align-items: flex-start;
  }

  .services-grid {
    grid-template-columns: 1fr;
  }
}
Code Explanation
@media (max-width: 768px) Yeh CSS tab apply hogi jab screen width 768px ya usse kam hogi.
flex-direction: column; Navbar items row ke bajay column mein aa jaate hain.
grid-template-columns: 1fr; Grid mobile par one-column layout ban jaati hai.

Chapter 9: Animations and Transitions

Transition style changes ko smooth banata hai. Animation repeated ya staged movement create karta hai. Beginners ke liye hover transitions sabse common practical use case hai.

.button {
  transition: transform 0.2s ease, background 0.2s ease;
}

.button:hover {
  background: #1d4ed8;
  transform: translateY(-2px);
}
Code Explanation
transition Background and movement change ko smooth banata hai.
.button:hover Mouse hover hone par button ka hover style apply hota hai.
transform: translateY(-2px); Button hover par thoda upar move hota hai.

Chapter 10: CSS Performance and Best Practices

Good CSS clean, reusable, and maintainable hota hai. CSS random likhne se future changes difficult ho jaate hain. Professional CSS mein naming clear hoti hai, repeated code kam hota hai, and mobile responsiveness test hoti hai.

Best Practice Explanation
External CSS use karo Styles separate file mein clean rehte hain.
Meaningful class names .navbar, .service-card, .hero jaise names readable hote hain.
Reusable styles Same style repeat karne ke bajay reusable classes banao.
Mobile test karo Responsive design verify hota hai.
