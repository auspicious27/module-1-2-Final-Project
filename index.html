1. CSS Introduction

CSS ka full form hai Cascading Style Sheets. CSS ka use HTML elements ko style karne ke liye hota hai. CSS se hum colors, fonts, spacing, borders, shadows, layout, hover effects, responsiveness, and animations control kar sakte hain.

Without CSS, website plain document jaisi dikhti hai. With CSS, wahi website professional and user-friendly dikhti hai. Real websites mein HTML and CSS dono saath kaam karte hain: HTML meaning deta hai, CSS appearance deta hai.

Types of CSS

Type Example Explanation
Inline CSS <h1 style="color: blue;"> CSS direct HTML tag ke andar likhte hain. Small testing ke liye okay, but professional project mein avoid karna better hai.
Internal CSS <style> h1 { color: blue; } </style> CSS same HTML file ke head section mein likhte hain. Single-page demo ke liye useful.
External CSS <link rel="stylesheet" href="styles.css"> CSS separate file mein likhte hain. Real projects ke liye best because code clean and reusable hota hai.

External CSS Practical

<link rel="stylesheet" href="styles.css">
Code Detailed Explanation
<link> External resource connect karne ka HTML tag. CSS file connect karne ke liye use hota hai.
rel="stylesheet" Browser ko batata hai ki linked file stylesheet hai. Matlab browser is file ko design rules ki tarah read karega.
href="styles.css" CSS file ka path. Agar CSS file same folder mein hai toh direct styles.css likhenge.

2. CSS Syntax

CSS syntax selector, property, and value se milkar banta hai. Selector batata hai kis element ko style karna hai. Property batati hai kya style change karna hai. Value batati hai us property ki setting kya hogi.

h1 {
  color: blue;
  font-size: 36px;
}

Syntax Breakdown

Part Explanation
h1 Selector hai. Page ke saare h1 headings ko target karta hai.
{ } Declaration block. CSS rules iske andar likhe jaate hain.
color Property hai. Text ka color change karti hai.
blue Value hai. Text blue color ka ho jayega.
font-size Property hai. Text ka size control karti hai.
36px Value hai. Heading ka size 36 pixels hoga.
; Semicolon CSS declaration ko end karta hai.

Practical Example

body {
  font-family: Arial, sans-serif;
  background: #f8fafc;
  color: #0f172a;
}

Yeh code complete page ka default font, background color, and text color set karta hai.

3. CSS Selectors

Selector decide karta hai ki CSS kis HTML element par apply hogi. Agar selector wrong hai, CSS likhne ke baad bhi style apply nahi hogi. Isliye selectors CSS ka foundation hain.

Selector Type Code Meaning
Element Selector p Page ke saare paragraph tags target karega.
Class Selector .card Jis element par class="card" hoga usko target karega.
ID Selector #hero Jis element par id="hero" hoga usko target karega.
Descendant Selector nav a Nav ke andar ke anchor links target karega.
Group Selector h1, h2, h3 Multiple elements par same style apply karega.
Pseudo-class a:hover Mouse hover hone par style apply karega.

Practical Code

.card {
  background: white;
}

nav a:hover {
  color: #2563eb;
}

Code Explanation

Code Explanation
.card Class selector. Cards ko style karne ke liye reusable class.
background: white; Card ka background white banata hai.
nav a:hover Navigation links par mouse hover hone par style apply karta hai.

4. Specificity

Specificity ka matlab CSS priority. Agar same element par multiple CSS rules apply ho rahe hain, toh browser decide karta hai kaunsa rule win karega. ID selector class selector se strong hota hai, class selector element selector se strong hota hai.

p {
  color: black;
}

.text {
  color: blue;
}

#intro {
  color: red;
}

Agar ek paragraph par class="text" and id="intro" dono hain, toh red color apply hoga because ID selector stronger hota hai.

Selector Priority
Inline Style Highest
ID Selector High
Class Selector Medium
Element Selector Low

5. Box Model and Layout Concepts

CSS box model ke according har HTML element ek box hota hai. Har box ke four parts hote hain: content, padding, border, and margin. Layout and spacing samajhne ke liye box model bahut important hai.

Part Meaning
Content Actual text/image area.
Padding Content and border ke beech inside space.
Border Element ke around line.
Margin Element ke bahar ka space.

Practical Code

.service-card {
  background: white;
  padding: 24px;
  border: 1px solid #e2e8f0;
  margin-bottom: 20px;
  border-radius: 8px;
}

Line-by-Line Explanation

Code Explanation
background: white; Card ka background white banata hai.
padding: 24px; Card ke andar content ke around space create karta hai.
border: 1px solid #e2e8f0; Card ke around light border create karta hai.
margin-bottom: 20px; Card ke neeche outside space add karta hai.
border-radius: 8px; Card ke corners rounded banata hai.

6. Flexbox Layout System

Flexbox one-dimensional layout system hai. Iska use row ya column direction mein items align karne ke liye hota hai. Navbar, buttons group, hero content alignment ke liye Flexbox bahut useful hai.

Practical Code

.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  gap: 24px;
}

Code Explanation

Code Explanation
display: flex; Container ke children ko flex layout mein arrange karta hai.
justify-content: space-between; Items ko horizontal direction mein left and right side spread karta hai.
align-items: center; Items ko vertical center align karta hai.
gap: 24px; Items ke beech fixed spacing add karta hai.

7. CSS Grid Layout System

CSS Grid two-dimensional layout system hai. Grid rows and columns dono control kar sakta hai. Cards, galleries, services sections, dashboards, and page layouts ke liye Grid powerful hai.

Practical Code

.services-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 24px;
}

Code Explanation

Code Explanation
display: grid; Container ko grid layout mein convert karta hai.
grid-template-columns: repeat(3, 1fr); 3 equal columns create karta hai. 1fr available space ka equal fraction hai.
gap: 24px; Cards ke beech spacing add karta hai.

8. Responsive Design Principles

Responsive design ka matlab website different screen sizes par properly work kare. Desktop, tablet, and mobile par website readable and usable honi chahiye. Responsive design ke liye flexible width, responsive images, readable typography, and media queries use hote hain.

Practical Code

img {
  max-width: 100%;
  height: auto;
}

.container {
  width: min(1120px, 92%);
  margin: 0 auto;
}

Code Explanation

Code Explanation
max-width: 100%; Image ko parent container se bahar overflow hone se prevent karta hai.
height: auto; Image ka original ratio maintain karta hai.
width: min(1120px, 92%); Desktop par max 1120px width, mobile par 92% width use karta hai.
margin: 0 auto; Container ko center align karta hai.

9. Media Queries

Media query CSS ka rule hai jo screen size ke according styles change karta hai. Example: desktop par 3 columns, mobile par 1 column. Responsive design mein media queries very important hain.

Practical Code

@media (max-width: 768px) {
  .navbar {
    flex-direction: column;
    align-items: flex-start;
  }

  .services-grid {
    grid-template-columns: 1fr;
  }
}

Code Explanation

Code Explanation
@media (max-width: 768px) Yeh CSS tab apply hogi jab screen width 768px ya usse kam hogi.
flex-direction: column; Navbar items row ke bajay column mein aa jaate hain.
align-items: flex-start; Navbar items left side align hote hain.
grid-template-columns: 1fr; Grid mobile par one-column layout ban jaati hai.

10. Animations and Transitions

Transition style changes ko smooth banata hai. Animation repeated or staged movement create karta hai. Beginners ke liye hover transitions sabse common practical use case hai.

Practical Code

.button {
  transition: transform 0.2s ease, background 0.2s ease;
}

.button:hover {
  background: #1d4ed8;
  transform: translateY(-2px);
}

Code Explanation

Code Explanation
transition Background and movement change ko smooth banata hai.
.button:hover Mouse hover hone par button ka hover style apply hota hai.
background: #1d4ed8; Hover par button ka background dark blue hota hai.
transform: translateY(-2px); Button hover par thoda upar move hota hai.

11. CSS Performance and Best Practices

Good CSS clean, reusable, and maintainable hota hai. CSS random likhne se future changes difficult ho jaate hain. Professional CSS mein naming clear hoti hai, repeated code kam hota hai, and mobile responsiveness test hoti hai.

Best Practice Explanation
External CSS use karo Styles separate file mein clean rehte hain.
Meaningful class names .navbar, .service-card, .hero jaise names readable hote hain.
Reusable styles Same card style repeat karne ke bajay reusable class banao.
Consistent colors Website professional dikhti hai.
Mobile test karo Responsive design verify hota hai.
Over-specific selectors avoid karo CSS maintain karna easy hota hai.
