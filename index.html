CSS Starting Explanation - Pehle CSS Ko Simple Way Mein Samjho

CSS ka full form hai Cascading Style Sheets. Simple words mein, CSS website ka design department hai. HTML page ka content and structure banata hai, jaise heading, paragraph, image, button, section, table. Lekin CSS decide karta hai ki woh content screen par kaisa dikhega: color kya hoga, text kitna bada hoga, button ka shape kaisa hoga, boxes ke beech gap kitna hoga, mobile par layout kaise adjust hoga, and hover karne par effect kya aayega.

Real-world example se samjho: HTML ek ghar ka structure hai, jisme rooms, doors, windows, and walls hote hain. CSS us ghar ka paint, lighting, furniture arrangement, decoration, and spacing hai. Structure ke bina ghar nahi banega, aur styling ke bina ghar plain and unfinished lagega. Isi tarah HTML ke bina website ka content nahi banega, aur CSS ke bina website professional nahi dikhegi.

Class mein CSS explain karte time pehle learners ko ek plain HTML page dikhao. Phir same HTML par CSS apply karke output dikhao. Jab woh dekhenge ki same content styling ke baad beautiful card, button, navbar, and layout ban gaya, toh unko CSS ka importance easily samajh aa jayega.

In Starting Demos Ko VS Code Mein Kaise Implement Karna Hai?

In 3 demos ko final project start karne se pehle short warm-up practice ki tarah karwana hai. Isse learners ko CSS ka basic flow samajh aa jayega: HTML mein class dena, CSS file connect karna, selector likhna, property-value apply karna, aur browser mein output check karna.

Step Action Simple Explanation
1 Desktop ya Documents mein css-demo-practice naam ka folder banao. Ye temporary practice folder hai. Isme hum sirf small CSS demos test karenge, final business website project nahi.

Demo 1: Plain Text Ko Styled Heading Banana

Is demo ka goal hai learners ko dikhana ki CSS ek simple heading ko attractive bana sakta hai. Pehle HTML heading normal dikhegi. CSS apply karne ke baad uska color, size, and spacing change ho jayega.

Demo 1 Implementation Files

Demo 1 run karne ke liye index.html mein complete basic HTML structure likho. Body ke andar sirf heading rakho. Phir style.css mein heading ki styling likho.

index.html

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>CSS Demo 1</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <h1 class="main-title">Welcome to My Website</h1>
</body>
</html>

style.css

.main-title {
  color: #2563eb;
  font-size: 36px;
  text-align: center;
  margin-top: 40px;
}

Expected Output: Browser mein ek heading show hogi: Welcome to My Website. Heading blue color mein hogi, center aligned hogi, size bada hoga, aur top se thoda gap hoga. Agar CSS file link nahi hui, toh heading default black and left aligned dikhegi.

Code Part Detailed Simple Hinglish Explanation
class="main-title" HTML heading ko ek class name diya gaya hai. Class ka use CSS mein specific element ko target karne ke liye hota hai. Yahan hum sirf is heading ko style karna chahte hain, isliye class use ki.
.main-title CSS mein dot . class selector ko represent karta hai. Matlab browser ko instruction milti hai ki jahan bhi HTML mein main-title class mile, wahan yeh styling apply karo.
color: #2563eb; Heading ka text color blue set karta hai. Hex color code professional websites mein commonly use hota hai because exact color control milta hai.
font-size: 36px; Heading ka size bada karta hai. px ka matlab pixels hota hai. Bigger font se heading page par important dikhti hai.
text-align: center; Heading ko center mein align karta hai. Landing page ya hero section mein main heading center align karna common design pattern hai.
margin-top: 40px; Heading ke upar space add karta hai. Margin outside spacing hoti hai. Isse heading top edge se chipki hui nahi dikhegi.

Demo 2: Simple Link Ko Button Banana

Is demo se learners ko samjhao ki HTML mein button jaisa dikhne wala element actual mein link bhi ho sakta hai. CSS us link ko button ka visual design deta hai.

Implementation: Isi css-demo-practice folder mein kaam karo. index.html ke body ke andar Demo 1 wali heading replace karke neeche wala link paste karo. Phir style.css ka old CSS remove karke Demo 2 wala CSS paste karo. Save karne ke baad browser refresh karo ya Live Server output dekho.

<a class="demo-button" href="#contact">Contact Us</a>
.demo-button {
  display: inline-block;
  background: #2563eb;
  color: white;
  padding: 12px 20px;
  border-radius: 8px;
  font-weight: 700;
}

Expected Output: Browser mein Contact Us ek normal underlined link ki tarah nahi, balki blue rounded button ki tarah dikhega. Button ke andar white text, proper spacing, and bold font hoga.

Code Part Detailed Simple Hinglish Explanation
<a> Anchor tag link create karta hai. Normally link blue underline ke saath dikhta hai. CSS ki help se same link ko button jaisa design de sakte hain.
display: inline-block; Anchor tag normally inline hota hai, isliye width, height, and padding ka behavior limited hota hai. inline-block se link inline bhi rehta hai and button jaisa box bhi ban jaata hai.
background: #2563eb; Button ka background blue karta hai. Background color button ko normal text link se alag and important banata hai.
color: white; Button ke andar text white karta hai, taaki blue background par text clearly readable ho.
padding: 12px 20px; Button ke andar space add karta hai. Pehla value top-bottom spacing hai, second value left-right spacing hai. Padding ke bina button tight and small dikhega.
border-radius: 8px; Button ke corners rounded karta hai. Rounded corners modern UI mein soft and friendly look dete hain.
font-weight: 700; Button text ko bold karta hai. Call-to-action button mein bold text user ka attention grab karta hai.

Demo 3: Hover Effect Se Interaction Dikhana

Hover effect ka matlab hai jab user mouse kisi element ke upar le jaata hai, tab style change hota hai. Yeh learner ko samjhata hai ki CSS sirf static design nahi, small interaction bhi create kar sakta hai.

Implementation: Demo 3 ke liye Demo 2 ka same HTML button rakho. style.css mein Demo 2 ka button CSS bhi rakho, aur uske neeche hover wala CSS add karo. Agar sirf hover CSS likhoge aur button ka base CSS remove kar doge, toh button ka full design nahi dikhega.

.demo-button:hover {
  background: #1d4ed8;
  transform: translateY(-2px);
}

Expected Output: Jab mouse button ke upar le jaoge, button ka color thoda dark blue ho jayega aur button halka sa upar move karega. Isse user ko feel hota hai ki element clickable hai.

Code Part Detailed Simple Hinglish Explanation
:hover :hover pseudo-class hai. Yeh tab apply hota hai jab user mouse element ke upar rakhta hai. Isse clickable elements interactive feel dete hain.
background: #1d4ed8; Hover par button ka blue color thoda dark ho jaata hai. User ko visual feedback milta hai ki button active/clickable hai.
transform: translateY(-2px); Button ko 2 pixels upar move karta hai. Negative Y value ka matlab upward movement hota hai. Isse button press/hover effect professional lagta hai.
Class demo idea: Pehle sirf HTML line run karo, phir CSS add karo, phir hover code add karo. Teen stages mein output dikhane se learners ko clear dikhega ki CSS styling step-by-step kaise page ko improve karti hai.

Code Part Detailed Simple Hinglish Explanation
<div class="info-card"> div ek generic container hota hai. Hum iske andar heading and paragraph rakh rahe hain. info-card class dene ka reason yeh hai ki CSS mein is complete block ko card jaisa design de sakein.

color: #1e293b; Card ke andar default text color dark slate set karta hai. Dark text light background par readable hota hai.

border-left: 6px solid #2563eb; Card ke left side par blue border add karta hai. Isse card important note jaisa dikhta hai. 6px border ki thickness hai, solid border style hai, and color blue hai.
.info-card h2 Yeh descendant selector hai. Matlab sirf woh h2 target hoga jo info-card ke andar hai. Page ke baaki h2 headings par yeh rule apply nahi hoga.

Simple Example 2: Box Model With Product Card

Box model CSS ka very important concept hai. Har HTML element ek box ki tarah behave karta hai. Us box ke andar content hota hai, content ke around padding hoti hai, padding ke bahar border hota hai, and border ke bahar margin hota hai.

<div class="product-card">
  <h3>Starter Website</h3>
  <p>A simple responsive website for small businesses.</p>
  <strong>Rs. 4999</strong>
</div>
.product-card {
  width: 280px;
  padding: 20px;
  margin: 30px auto;
  border: 1px solid #dbeafe;
  border-radius: 12px;
  box-shadow: 0 10px 25px rgba(15, 23, 42, 0.10);
}
Code Part Detailed Simple Hinglish Explanation
width: 280px; Card ki fixed width 280 pixels set karta hai. Width se card ka size control hota hai. Agar width na dein, toh block element full row width le sakta hai.
padding: 20px; Card ke andar content aur border ke beech space deta hai. Yeh box model ka inner space hai. Padding content ko breathing room deti hai.
margin: 30px auto; Card ke bahar space set karta hai. 30px top-bottom margin hai. auto left-right margin card ko center align karta hai, jab width fixed ho.
border: 1px solid #dbeafe; Card ke around thin border banata hai. Border card boundary ko visually clear karta hai.
border-radius: 12px; Card ke corners rounded karta hai. Modern card UI mein rounded corners commonly use hote hain.
box-shadow Card ke neeche soft shadow add karta hai. Shadow se card page se thoda lifted dikhta hai. Yeh depth create karta hai.

Simple Example 3: Flexbox Se Items Side-by-Side Arrange Karna

Flexbox ka use tab hota hai jab hume items ko ek row ya column mein neatly align karna ho. Navbar, buttons group, profile row, and small cards alignment ke liye Flexbox bahut useful hai.

<div class="feature-row">
  <div>Fast</div>
  <div>Responsive</div>
  <div>SEO Friendly</div>
</div>
.feature-row {
  display: flex;
  gap: 16px;
  justify-content: center;
  align-items: center;
}

.feature-row div {
  background: #f8fafc;
  border: 1px solid #e2e8f0;
  padding: 14px 18px;
  border-radius: 8px;
}
Code Part Detailed Simple Hinglish Explanation
display: flex; Parent container ko Flexbox container banata hai. Iske andar ke child divs default row direction mein side-by-side aa jaate hain.
gap: 16px; Flex items ke beech equal spacing add karta hai. Gap use karna margin manually dene se cleaner hota hai.
justify-content: center; Items ko horizontal direction mein center karta hai. Agar row mein extra space ho, toh items center area mein grouped dikhenge.
align-items: center; Items ko vertical direction mein center align karta hai. Navbar mein logo and links ko same height feel dene ke liye yeh property useful hoti hai.
.feature-row div Yeh selector feature row ke andar ke child divs ko target karta hai. Isse har feature item ko small card/badge design milta hai.

HTML Code Detailed Explanation

Code Detailed Explanation
<link rel="stylesheet" href="styles.css"> Yeh line HTML file ko external CSS file se connect karti hai. Browser jab index.html load karta hai, toh is line ki wajah se styles.css file bhi load hoti hai. Agar yeh line missing ho, ya file name wrong ho, toh HTML content browser mein dikh jaayega but styling apply nahi hogi. Isliye CSS project ka first checkpoint yahi line hoti hai.
class="container navbar" Same element par two classes use ki gayi hain. container class page content ko fixed readable width mein center rakhegi. navbar class usi div ke andar Flexbox layout apply karegi, jisse logo and navigation links proper row mein align honge. Ek element par multiple classes dena common professional practice hai.
class="nav-links" Yeh class navigation links ke group ko target karne ke liye hai. Iske andar Home, Services, About, Contact links hain. CSS mein .nav-links se links ko row mein arrange karenge, gap denge, and .nav-links a:hover se hover effect add karenge. Isse navigation clean and interactive lagega.
class="hero" Hero website ka first main visual section hota hai. User page open karte hi sabse pehle isi section ko dekhta hai. Is project mein hero section business ka main message, supporting text, button, and feature card dikhata hai. CSS se is section ko large padding, gradient background, and professional spacing milegi.
class="hero-grid" Hero section ke andar content ko Grid layout mein arrange karne ke liye yeh class use hoti hai. Left side par heading, paragraph, and button hoga. Right side par hero-card hoga. CSS Grid ki help se desktop par two-column layout banega, aur media query ki help se mobile par one-column layout ban jayega.
class="button" HTML mein yeh actual anchor link hai, but CSS se isko button jaisa design diya jayega. button class background color, white text, padding, border-radius, font-weight, and hover transition apply karegi. Isse call-to-action clear dikhega and user ko click karne ka signal milega.
class="services-grid" Services section ke cards ko CSS Grid se columns mein arrange karne ke liye yeh wrapper class hai. Desktop par 3 service cards ek row mein dikhenge. Mobile par media query grid-template-columns: 1fr; apply karegi, jisse cards one-by-one stack honge.
class="service-card" Har service item ko card design dene ke liye yeh class use hoti hai. CSS se card ko white background, border, padding, rounded corners, and box-shadow milega. Hover par card thoda upar move hoga, jisse interactive and modern UI feel aayega.

CSS Code Detailed Explanation

CSS Code Detailed Explanation
* { box-sizing: border-box; } Universal selector page ke saare elements par apply hota hai. box-sizing: border-box; ka meaning hai ki element ki width calculate karte waqt padding and border bhi usi width ke andar count honge. Isse layout predictable hota hai. Beginner ke liye yeh important hai kyunki padding add karne par box unexpected bada nahi hota.
body body complete webpage ko represent karta hai. Is rule ke andar hum page ka default margin, font, text color, background color, and line-height set karte hain. Body styling ek base layer jaisi hoti hai. Agar body clean set ho, toh baaki sections ko style karna easy ho jaata hai.
a Yeh selector page ke saare anchor links ko target karta hai. Default browser links blue and underlined hote hain. Business website mein hum links ko custom design dena chahte hain, isliye color: inherit; and text-decoration: none; use karte hain. Isse nav links and buttons clean dikhte hain.
.container .container website ke content ko center and readable width mein rakhta hai. Large desktop screens par content full width mein stretch nahi hota. Mobile par 92% width use hoti hai, jisse left-right side mein thoda safe space bachta hai. Professional websites mein container class bahut common hoti hai.
.site-header Header website ka top navigation area hai. .site-header rule usko white background, bottom border, and sticky position deta hai. Sticky position ka benefit yeh hai ki scroll karte waqt header top par visible reh sakta hai, jisse navigation easily accessible hoti hai.
.navbar .navbar Flexbox layout apply karta hai. Iske andar logo and navigation links hain. Flexbox ki help se logo left side and links right side align hote hain. align-items: center; se dono vertically balanced dikhte hain.
justify-content: space-between; Yeh Flexbox property navbar ke items ko horizontal direction mein spread karti hai. Logo left side rahega and nav links right side chale jayenge. Isse navbar professional and balanced dikhta hai.
.nav-links Navigation links ke wrapper ko target karta hai. display: flex; se links row mein aate hain and gap: 18px; se links ke beech consistent spacing aati hai. Spacing ke bina links chipke hue dikhenge.
.nav-links a:hover Yeh pseudo-class selector hai. Jab user mouse navigation link par le jaata hai, tab yeh rule apply hota hai. Link ka color blue ho jaata hai. Hover effect user ko batata hai ki yeh item clickable hai.
.hero Hero section website ka main introduction area hai. padding: 80px 0; se top and bottom mein large space milta hai. Gradient background se section visually attractive dikhta hai. Hero ka goal hai user ko immediately business ka main message samjhana.
.hero-grid Hero content ko CSS Grid se two columns mein divide karta hai. 1.3fr left content ko zyada space deta hai and 0.7fr right card ko thoda kam space deta hai. Yeh layout desktop par modern landing page jaisa dikhta hai.
.button Anchor link ko button design deta hai. Background blue, text white, padding, border-radius, and bold font milta hai. display: inline-block; important hai kyunki anchor normally inline element hota hai; inline-block se padding and box shape properly kaam karta hai.
transition Transition hover effect ko smooth banata hai. Without transition, background and movement sudden change honge. With transition, button ka color and movement soft feel deta hai. Yeh small detail website ko polished banati hai.
.hero-card, .service-card, .about-box, .contact-box Yeh group selector hai. Multiple classes ko comma se ek saath target kiya gaya hai. Iska benefit hai ki same card styling baar-baar repeat nahi karni padti. Hero card, service cards, about box, and contact box sabko consistent white card design milta hai.
.services-grid Services cards ka parent grid container hai. display: grid; and repeat(3, 1fr) se desktop par 3 equal columns bante hain. Yeh CSS Grid topic ka practical use hai.
@media (max-width: 768px) Media query responsive design ke liye use hoti hai. Iska meaning hai: jab screen width 768px ya usse kam ho, tab andar ke CSS rules apply karo. Mobile/tablet par layout adjust karne ke liye yeh very important hai.
grid-template-columns: 1fr; Mobile screen par Grid ko one-column layout mein convert karta hai. Desktop par hero/services side-by-side ho sakte hain, but mobile par side-by-side layout cramped dikhega. Isliye one-column layout readable and user-friendly hota hai.

Is project ka purpose hai ki learners samjhein ki real website mein HTML and CSS ka relation kaise hota hai. HTML content rakhta hai: header, menu cards, about text, contact text. CSS design deta hai: colors, spacing, navbar, cards, button, grid layout, and mobile responsive layout.

index.html Code Explanation - Har Line Ka Matlab

HTML Line / Code Detailed Simple Hinglish Explanation
<!DOCTYPE html> Browser ko batata hai ki yeh HTML5 document hai. Isko file ke top par likhna best practice hai.
<html lang="en"> Complete HTML page ka root element start hota hai. lang="en" browser and screen readers ko batata hai ki page ki main language English hai.
<head> Head section page ki hidden information rakhta hai, jaise title, metadata, and CSS file link. Yeh content direct page body mein visible nahi hota.
<meta charset="UTF-8"> Text encoding set karta hai. Isse letters, symbols, and special characters properly show hote hain.
<meta name="viewport" content="width=device-width, initial-scale=1.0"> Mobile responsive behavior ke liye important line hai. Browser ko bolti hai ki page device width ke according render karo.
<title>FreshBite Cafe</title> Browser tab ka title set karta hai. Jab page open hoga toh tab mein FreshBite Cafe dikhega.
<link rel="stylesheet" href="styles.css"> HTML file ko CSS file se connect karta hai. href="styles.css" ka matlab CSS file same folder mein hai.
<body> Visible website content yahan se start hota hai. Browser screen par jo kuch dikhta hai, mostly body ke andar hota hai.
<header class="site-header"> Website ka top header start hota hai. site-header class CSS mein header ko background, border, and sticky behavior dene ke liye use hogi.
<div class="container navbar"> Ek div par two classes hain. container width control karega and navbar logo and links ko Flexbox se arrange karega.
<h1 class="logo">FreshBite Cafe</h1> Website ka main brand name hai. logo class se color and font size set hoga.
<nav class="nav-links"> Navigation area start hota hai. Iske andar page ke sections par jaane wale links rahenge.
<a href="#menu">Menu</a> Menu section par scroll karne wala link hai. #menu same page ke id="menu" section ko target karta hai.
<a href="#about">About</a> About section par le jaane wala link hai. Isse one-page website navigation easy hoti hai.
<a href="#contact">Contact</a> Contact section par le jaane wala link hai. User quickly contact information tak pahunch sakta hai.
<main> Page ka main content start hota hai. Header and footer ke beech ka primary content main ke andar rakha jata hai.
<section class="hero"> Hero section website ka first important section hai. Isme main message and button hota hai.
<div class="container hero-content"> Hero content ko center and readable width mein rakhta hai. hero-content text alignment and max width ke liye use hota hai.
<p class="tagline">Fresh food, friendly place</p> Short label/tagline hai. CSS se uppercase, orange color, and letter spacing milegi.
<h2>Healthy meals made simple for busy people.</h2> Hero section ki main heading hai. User ko immediately cafe ka main value samajh aata hai.
<p class="hero-text">...</p> Hero heading ke neeche supporting paragraph hai. CSS se text soft gray and slightly bigger hoga.
<a class="button" href="#menu">View Menu</a> Call-to-action link hai. CSS isko orange rounded button banayegi. Click karne par menu section par jayega.
<section id="menu" class="section"> Menu section start hota hai. id="menu" navigation link ke liye hai. section class spacing ke liye hai.
<div class="menu-grid"> Menu cards ka wrapper hai. CSS Grid se 3 cards ek row mein arrange honge.
<article class="menu-card"> Har menu item ek independent card hai. Article tag meaningful hai kyunki har card apne aap mein complete information block hai.
<h3>Veg Sandwich</h3> Menu item ka title hai. Same pattern Fruit Bowl and Cold Coffee cards mein bhi use hota hai.
<p>Fresh vegetables...</p> Menu item ka short description hai. User ko item ke baare mein simple information milti hai.
<strong>Rs. 99</strong> Price ko important dikhane ke liye strong tag use kiya hai. CSS se price orange and bigger dikhega.
<section id="about" class="section about-box"> About section start hota hai. about-box class se white background apply hoga.
<section id="contact" class="section contact-box"> Contact section start hota hai. Isme timing and email button hai. contact-box text center karega.
<a class="button" href="mailto:hello@freshbite.com">Email Us</a> Email link hai. Click karne par user's mail app open ho sakta hai. Same button class se design consistent rahega.
<footer class="site-footer"> Website ka bottom area hai. Isme small project credit text rakha gaya hai.

styles.css Code Explanation - Har CSS Rule Ka Matlab

CSS Code Detailed Simple Hinglish Explanation
* { box-sizing: border-box; } Universal selector saare elements par apply hota hai. border-box layout calculation easy karta hai because padding and border element ki width ke andar count hote hain.
body Complete page ke base styles set karta hai: margin remove, font set, text color, background color, and line height.
margin: 0; Browser ka default body margin remove karta hai. Isse header and sections screen edge se clean start hote hain.
font-family: Arial, Helvetica, sans-serif; Page ka font set karta hai. Agar Arial available nahi hai toh Helvetica use hogi, warna generic sans-serif use hoga.
background: #fff7ed; Complete page ka light warm background set karta hai. Cafe theme ke liye soft orange background suitable hai.
a All links ko target karta hai. Default underline remove and text color inherit karne ke liye use hota hai.
.container Content ko center and fixed readable width mein rakhta hai. Large screen par content zyada spread nahi hota.
width: 90%; Container screen ka 90% width lega. Isse mobile and desktop dono par side spacing bachi rahegi.
max-width: 1100px; Container ko very large screen par bhi 1100px se zyada wide nahi hone deta. Readability better hoti hai.
.site-header Top header ko style karta hai. White background, bottom border, and sticky behavior isi rule se milta hai.
position: sticky; top: 0; Header ko scroll ke time top par stick karne mein help karta hai. top: 0 batata hai ki header top edge par rahe.
.navbar Logo and navigation links ko Flexbox se arrange karta hai. Desktop par logo left and links right side rahenge.
display: flex; Element ko Flexbox container banata hai. Iske child elements row mein align ho sakte hain.
align-items: center; Navbar items ko vertical center align karta hai, taaki logo and links balanced dikhein.
justify-content: space-between; First item ko left and last group ko right side push karta hai. Navbar professional dikhta hai.
.logo Cafe name ko orange color and proper font size deta hai. Logo brand identity clear karta hai.
.nav-links Navigation links ko row mein arrange karta hai and links ke beech gap add karta hai.
.nav-links a:hover Mouse hover par nav link ka color orange ho jaata hai. User ko clickable feedback milta hai.
.hero Hero section ko light gradient background and large top-bottom spacing deta hai.
.hero-content Hero text ko limited width and center align karta hai. Isse message clean and focused dikhta hai.
.tagline Small intro text ko orange, bold, uppercase, and spaced letters ke saath highlight karta hai.
.hero h2 Hero heading ka large font size and line height set karta hai. Heading page ka main attention point hoti hai.
.hero-text Supporting paragraph ko gray color and readable size deta hai. Yeh heading ko explain karta hai.
.button Normal link ko button jaisa design deta hai: orange background, white text, padding, rounded shape, bold text.
.button:hover Hover par button dark orange ho jaata hai. Isse button interactive feel deta hai.
.section Common section spacing set karta hai. Menu, about, contact sections ko top-bottom padding milti hai.
.section h2 Section headings ko center and bigger size mein show karta hai. Har section ka title clearly visible hota hai.
.menu-grid Menu cards ka Grid layout parent hai. Desktop par cards 3 columns mein arrange honge.
grid-template-columns: repeat(3, 1fr); 3 equal columns banata hai. 1fr ka matlab available space ka equal fraction.
gap: 20px; Cards ke beech equal gap add karta hai. Gap ke bina cards chipke hue dikhenge.
.menu-card Har menu item ko card style deta hai: white background, border, rounded corners, padding, and shadow.
box-shadow Card ke neeche soft shadow add karta hai. Card page se slightly lifted feel hota hai.
.about-box About section ko white background deta hai, jisse section page background se separate dikhta hai.
.about-box p, .contact-box p Group selector hai. About and contact paragraphs ko same max width and center alignment deta hai.
.contact-box Contact section ka text center align karta hai. Contact CTA usually center aligned achha dikhta hai.
.site-footer Footer ko dark background, white text, center alignment, and padding deta hai.
@media (max-width: 768px) Responsive design rule start karta hai. Jab screen 768px ya usse kam hogi, andar ke styles apply honge.
flex-direction: column; Mobile par navbar items ko vertical stack karta hai. Small screen par links fit ho jaate hain.

.menu-grid { grid-template-columns: 1fr; } Mobile par menu cards ko one-column layout mein convert karta hai. Cards one-by-one readable dikhenge.
